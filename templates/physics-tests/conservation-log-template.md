This template extends the Tier 3 physics property tests with a per-step diagnostic for conservation laws. The pattern: choose an integrator with appropriate conservation properties, log a conserved quantity at every step, plot drift over time, and assert that drift remains bounded (not monotonic).

## When to use this template

- Your simulation contains a Hamiltonian (or near-Hamiltonian) system.
- You run for many oscillation periods.
- You care about long-time qualitative behaviour, not just short-time accuracy.
- You suspect or want to rule out integrator-induced energy drift.

For ordinary short integrations (less than ~$Q$ periods), RK4 is fine and this template is overkill. For long integrations (planetary motion, molecular dynamics, plasma simulations, slow-decay-time pendulum studies), it becomes essential.

## Why bounded drift is the right metric

A *symplectic* integrator (Velocity Verlet, leapfrog, Yoshida 4th-order, Forest-Ruth) preserves the symplectic 2-form on phase space. Energy is not conserved exactly — but the energy error oscillates around the true value with bounded amplitude over exponentially long times. This is a theorem (backward error analysis; see Hairer-Lubich-Wanner Ch. IX).

A non-symplectic integrator (RK4, Euler) has *secular* energy drift: the error grows roughly linearly with time. Over $10^4$ periods, an RK4-integrated pendulum visibly loses or gains energy.

The diagnostic is the *character* of the drift, not its magnitude:

- **Bounded oscillation, no trend** → integrator is symplectic and correctly implemented; ✓
- **Monotonic drift** → either a non-symplectic integrator is being used (use a symplectic one), or the implementation is buggy
- **Sudden jumps or NaN** → numerical instability; reduce $dt$ or change method

## Choosing an integrator

| Integrator | Order | Symplectic? | Use case |
|---|---|---|---|
| Forward Euler | 1 | No | Never (unstable for oscillators) |
| RK4 | 4 | No | Short integrations, non-Hamiltonian systems |
| Velocity Verlet | 2 | Yes | Workhorse for molecular dynamics, simple Hamiltonian systems |
| Leapfrog | 2 | Yes | Same as Velocity Verlet, different staggered form |
| Yoshida 4th-order | 4 | Yes | Long-time accuracy + symplectic; standard for celestial mechanics |
| Forest-Ruth | 4 | Yes | Like Yoshida 4 with smaller error coefficient |

For a driven-pendulum-like system (slowly decaying oscillator with periodic forcing), Velocity Verlet on the conservative part + a small explicit treatment of the dissipation gives good long-time behaviour. For purely undamped systems, Yoshida 4th-order is the standard.

```python
"""test_conservation_log.py — Tier 3 extension.

Logs a conserved quantity at every step of a long integration,
asserts bounded drift, and writes a diagnostic plot to
tests/physics/reports/.

If a test fails, READ THE PLOT before making code changes.
The shape of the drift (bounded vs monotonic) is the diagnostic."""
import os
import numpy as np
import pytest
import matplotlib.pyplot as plt


def test_undamped_long_run_energy_bounded(integrator, energy, standard_params):
    """Long run of undamped pendulum — energy must remain bounded.
    
    Tier 3. Catches: integrator-induced secular energy drift,
    accidentally non-symplectic implementation when symplectic was claimed.
    
    Note: this test uses a long t_end (1000 periods). With RK4 it WILL
    show drift; that's expected for RK4. Adapt the assertion to your
    integrator's properties (bounded for symplectic; bounded ratio for RK4
    at small dt; monotonic = bug for symplectic)."""
    from sim.pendulum import PendulumState
    standard_params.Q = 1e12  # undamped
    initial = PendulumState(theta=np.radians(5.0), theta_dot=0.0)
    
    n_periods = 1000
    t_end = n_periods * standard_params.T_0
    dt = 1e-3
    
    traj = integrator(state=initial, t_end=t_end, dt=dt)
    
    # Compute energy at every step (vectorised)
    E_t = energy(traj["theta"], traj["theta_dot"])
    E_0 = E_t[0]
    relative_drift = (E_t - E_0) / E_0
    
    # --- Diagnostic plot ---
    fig, ax = plt.subplots(figsize=(8, 4))
    ax.plot(traj["t"], relative_drift)
    ax.set_xlabel("t [s]")
    ax.set_ylabel(r"$(E(t) - E_0) / E_0$")
    ax.set_title(f"Energy drift over {n_periods} periods (dt={dt})")
    ax.grid(True, alpha=0.3)
    
    out_dir = os.path.join(os.path.dirname(__file__), "reports")
    os.makedirs(out_dir, exist_ok=True)
    plot_path = os.path.join(out_dir, "energy_drift_undamped.png")
    fig.savefig(plot_path, dpi=120, bbox_inches='tight')
    plt.close(fig)
    
    # --- Assertions (adapt to your integrator) ---
    
    # Magnitude bound (loose; mainly catches catastrophic failure)
    assert np.max(np.abs(relative_drift)) < 0.1, (
        f"Energy drift magnitude exceeds 10% over {n_periods} periods.\n"
        f"See plot: {plot_path}"
    )
    
    # Character: monotonic drift is bad for a symplectic claim
    # Detect: fit linear trend to drift; check slope
    slope, intercept = np.polyfit(traj["t"], relative_drift, 1)
    
    # For a symplectic integrator, slope should be ~0 (drift bounded, not secular)
    # For RK4, expect a small non-zero slope (acceptable but worth tracking)
    SYMPLECTIC_INTEGRATOR = False  # set True if you adopt Velocity Verlet etc.
    if SYMPLECTIC_INTEGRATOR:
        assert abs(slope) < 1e-9, (
            f"Symplectic integrator showing secular drift (slope={slope:.2e}/s).\n"
            f"Either the integrator is not actually symplectic, or there is a bug.\n"
            f"See plot: {plot_path}"
        )
```

## Reading the diagnostic plot

The plot of $(E(t) - E_0)/E_0$ over time tells you which class of behaviour you have:

- **Flat line at zero (within noise)** — exact conservation; rare and usually means you have an analytical solution.
- **Bounded oscillation** — symplectic integrator working correctly. The amplitude of oscillation depends on $dt$ and the integrator; for Velocity Verlet at $dt = 10^{-3}$ on a unit-frequency oscillator, expect amplitude ~$10^{-6}$.
- **Linear monotonic drift** — non-symplectic integrator (e.g., RK4) accumulating error. Expected for RK4; not expected for Velocity Verlet. Slope tells you energy gained or lost per unit time.
- **Slowly accelerating drift (super-linear)** — usually a sign of growing instability; reduce $dt$ or change integrator.
- **Sudden jumps** — discrete events (e.g., escapement impulse) with timing error. Check whether the jump aligns with expected event times.
- **NaN after some time** — numerical blow-up; reduce $dt$, check for a divide-by-zero or similar.

## Limitations

- **Symplectic integrators don't compose with arbitrary external forces.** For periodic external forcing with discrete events (like the EM pulses in driven-pendulum), the symplecticity argument needs care. Consult Hairer-Lubich-Wanner Ch. IX for the right operator-splitting approach.
- **Damping breaks symplecticity.** If your system has any dissipation, no integrator is symplectic. The diagnostic then becomes "does the energy decay match the analytical $\exp(-2\beta t)$?" rather than "is the drift bounded?"
- **Plot inspection is required.** The bounded-vs-monotonic distinction is qualitative and best read from the plot. Automated assertion on slope can flag obvious cases but misses subtle ones. Treat the plot as the primary artefact.
- **Cost.** Long-run integration is expensive. Run this test less often than the standard suite — perhaps weekly CI, not every commit.
